# PostgreSQL Setup for Render Deployment

## What Changed

I've configured your ControlHub application to use PostgreSQL on Render (production) while keeping SQLite for local development.

### Files Updated:

1. **requirements.txt** - Added PostgreSQL dependencies:
   - `psycopg2-binary==2.9.9` - PostgreSQL adapter for Python
   - `dj-database-url==2.1.0` - Database URL parser for Django

2. **controlhub/settings.py** - Updated database configuration:
   - Automatically uses PostgreSQL when `DATABASE_URL` is present (Render)
   - Falls back to SQLite for local development
   - Added connection pooling and health checks

3. **render.yaml** - Added PostgreSQL database:
   - Created `controlhub-db` PostgreSQL database
   - Automatically links `DATABASE_URL` to your web service
   - Free tier PostgreSQL (1GB storage, expires after 90 days)

## Deployment Steps

### Option 1: Deploy via render.yaml (Recommended)

1. **Commit and push your changes:**
   ```bash
   git add .
   git commit -m "Add PostgreSQL support for Render deployment"
   git push origin main
   ```

2. **Deploy to Render:**
   - Go to [Render Dashboard](https://dashboard.render.com/)
   - Click "New +" → "Blueprint"
   - Connect your repository
   - Render will automatically:
     - Create the PostgreSQL database
     - Create the web service
     - Link them together
     - Run migrations
     - Create the superuser

3. **Wait for deployment** (usually 5-10 minutes for first deploy)

### Option 2: Manual Setup (If you already have a service)

1. **Create PostgreSQL Database:**
   - Go to Render Dashboard
   - Click "New +" → "PostgreSQL"
   - Name: `controlhub-db`
   - Database: `controlhub`
   - User: `controlhub`
   - Region: Oregon
   - Plan: Free
   - Click "Create Database"

2. **Link Database to Web Service:**
   - Go to your `controlhub` web service
   - Click "Environment"
   - Add new environment variable:
     - Key: `DATABASE_URL`
     - Value: Copy the "Internal Database URL" from your PostgreSQL database
   - Save changes

3. **Update Start Command:**
   - In your web service settings
   - Set Start Command to:
     ```
     daphne -b 0.0.0.0 -p $PORT controlhub.asgi:application
     ```

4. **Trigger Manual Deploy:**
   - Click "Manual Deploy" → "Deploy latest commit"

## Troubleshooting

### Database Connection Errors
- Verify `DATABASE_URL` is set in Render environment variables
- Check PostgreSQL database is running (green status)
- Ensure database and web service are in the same region

### Migration Errors
- Check build logs for specific error messages
- Ensure all migrations are committed to git
- Try manual migration: Add to build command temporarily:
  ```
  python manage.py migrate --run-syncdb
  ```

### Superuser Not Created
- Check build logs for errors
- Manually create via Render shell:
  ```bash
  python manage.py createsuperuser
  ```

## PostgreSQL Free Tier Limits

- **Expires:** 90 days (then you need to upgrade or recreate)
- **Connections:** Limited
- **Backups:** Not included in free tier
